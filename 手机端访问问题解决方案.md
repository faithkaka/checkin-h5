# 📱 手机端访问问题解决方案

## 🔍 问题现象
- ✅ 电脑浏览器能打开：`https://faithkaka.github.io/checkin-h5/`
- ❌ 手机浏览器打不开页面

## 🎯 根本原因

### 1. **Supabase 被墙**（最可能）
- Supabase 的 CDN (`cdn.jsdelivr.net`) 在中国大陆访问不稳定
- Supabase 后端服务 (`ussvekkgyntubivhfext.supabase.co`) 被防火墙拦截
- 手机网络（4G/5G/公司 WiFi）通常没有代理，直接访问会超时

### 2. **GitHub Pages 被干扰**
- 某些网络环境下，`github.io` 域名会被 DNS 污染
- HTTPS 连接可能被中间人拦截

### 3. **资源加载超时**
- 地图图片 `images/map-bg.png` 加载失败
- CSS/JS 文件引用路径问题

---

## ✅ 解决方案

### 方案一：使用国内 CDN 镜像（已应用）

**修改内容：**
```html
<!-- 原来 -->
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

<!-- 现在 -->
<script src="https://fastly.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
```

**优点：**
- fastly.jsdelivr.net 在国内访问速度更快
- 保留了 Supabase 数据同步功能

**缺点：**
- 仍然依赖被墙的 Supabase 后端服务

---

### 方案二：完全移除 Supabase（推荐用于纯展示）

如果只需要**本地打卡展示**，不需要云端同步：

**修改 index.html：**
```html
<!-- 移除 Supabase，只保留 app.js -->
<script src="js/app.js"></script>
```

**修改 app.js：**
应用已内置容错处理，如果 `SupabaseManager` 未加载，会自动使用：
- `localStorage` 本地存储打卡数据
- 临时用户 ID：`unknown_xxx`

**优点：**
- 完全不需要访问被墙的服务
- 加载速度极快
- 手机/电脑都能正常访问

**缺点：**
- 数据只保存在本地，清除缓存会丢失
- 无法跨设备同步数据

---

### 方案三：使用国内后端服务

如果需要数据同步，可以迁移到国内服务：

1. **阿里云 EMAS Serverless**
2. **腾讯云云开发**
3. **LeanCloud**

## 🔧 当前代码的容错机制

`app.js` 已经实现了优雅的降级处理：

```javascript
// 如果 SupabaseManager 加载失败
if (window.SupabaseManager) {
  await SupabaseManager.init();
  AppState.userId = SupabaseManager.userId;
} else {
  // 自动降级到本地存储模式
  AppState.userId = 'unknown_' + Date.now();
  console.warn('⚠️ SupabaseManager 未加载，使用本地存储');
}
```

## 🎯 最佳实践建议

### 对于打卡活动 H5：

1. **使用本地存储模式**（推荐）
   - 移除 Supabase 依赖
   - 数据保存在用户浏览器
   - 完全不受网络限制

2. **添加离线功能**
   ```html
   <link rel="manifest" href="manifest.json">
   ```

3. **优化首屏加载**
   - 内联关键 CSS
   - 压缩图片
   - Lazy load 非关键资源

4. **添加错误提示**
   ```javascript
   window.addEventListener('error', function(e) {
     alert('资源加载失败，请检查网络连接');
   });
   ```
