# 📱 支付宝多用户打卡 - 使用说明

## ✅ 功能概述

已实现自动区分不同支付宝用户的打卡数据，每个用户独立的：
- 📊 积分累计
- 📍 打卡记录
- 🏆 成就状态
- 🎫 兑奖券

## 🔧 实现原理

### 1. 用户识别流程

```
1. 检测支付宝环境 (AlipayClient/AlipayJSBridge)
   ↓
2. 获取用户 ID（优先级）:
   - URL 参数：?userId=xxx
   - 支付宝 AuthCode（需要授权）
   - localStorage 缓存
   - 生成临时 ID
   ↓
3. 数据存储键：alipay_{userId}_checkin_data
   ↓
4. 每个用户数据完全隔离
```

### 2. 用户 ID 来源

| 来源 | 格式 | 说明 |
|------|------|------|
| URL 参数 | `u_xxx` | 从外部传入 |
| 支付宝授权 | `alipay_xxx` | 需要用户授权 |
| 匿名用户 | `alipay_anonymous_xxx` | 未授权 |
| 游客 | `guest_xxx` | 非支付宝环境 |

## 🚀 部署方式

### 方式 1: 从支付宝打开（推荐）

在支付宝中打开 H5 页面：
```
https://yourdomain.com/checkin-h5/index.html
```

系统会自动：
- ✅ 检测支付宝环境
- ✅ 尝试获取用户 AuthCode
- ✅ 为用户分配唯一 ID
- ✅ 独立存储打卡数据

### 方式 2: 通过链接传入用户 ID

从支付宝或其他渠道传入用户 ID:
```
https://yourdomain.com/checkin-h5/index.html?userId=ALIPAY_USER_ID
```

支持的参数名：
- `userId`
- `alipayUserId`
- `uid`

### 方式 3: 打卡链接（带用户 ID）

```
https://yourdomain.com/checkin-h5/index.html?userId=ALIPAY_USER_ID&checkin=1
```

## 🔐 支付宝授权

### 获取用户真实 ID

需要在支付宝开放平台配置：

1. **小程序/生活号**
   ```javascript
   my.getAuthCode({
     scopes: ['auth_user']
   }, (res) => {
     // res.authCode 用于换取用户信息
   });
   ```

2. **H5 页面**
   ```javascript
   AlipayJSBridge.call('getAuthCode', {
     scopes: ['auth_user']
   }, callback);
   ```

### 注意事项

⚠️ **获取真实用户信息需要**：
- 支付宝开放平台账号
- 配置 OAuth 2.0 授权
- 用户授权同意

📝 **当前方案**：
- 使用 AuthCode 作为用户标识
- 或从 URL 传入用户 ID
- 保证数据隔离即可

## ⚠️ 注意事项

1. **数据持久化**
   - 数据存储在 localStorage
   - 清除浏览器缓存会丢失数据
   - 建议对接后端 API

2. **用户隐私**
   - 不要存储敏感信息
   - 遵循支付宝隐私政策
   - 明确告知用户数据用途

3. **跨设备**
   - localStorage 仅限单设备
   - 跨设备需要后端支持
   - 建议用户登录同步

4. **支付宝环境检测**
   - 非支付宝环境会使用游客模式
   - 建议提供降级方案
   - 微信/QQ 等环境同理
