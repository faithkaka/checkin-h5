# 📱 支付宝多用户打卡 - 使用说明

## ✅ 核心功能

**所有用户使用同一个链接**，系统自动识别不同支付宝账号，实现：
- 🔐 自动获取支付宝用户唯一标识
- 📊 每个用户独立的打卡数据
- ☁️ 数据存储在 Supabase 云端
- 📱 跨设备同步（同一账号）

## 🔗 统一打卡链接

### 局域网测试
```
http://30.230.99.236:9000/index.html
```

### 生产环境（示例）
```
https://yourdomain.com/index.html
```

**不需要在链接中传递用户 ID！** 系统会自动识别。

## 🚀 在支付宝中打开

### 方法 1: 扫码打开（推荐）

1. 用电脑浏览器打开：`http://30.230.99.236:9000`
2. 使用在线二维码工具生成二维码
   - 推荐：https://www.qr-code-generator.com/
   - 或使用浏览器的"创建二维码"功能
3. 打开支付宝 → 扫一扫
4. 扫描屏幕上的二维码
5. 在支付宝内打开链接

### 方法 2: 发送链接到支付宝

1. 复制链接：`http://30.230.99.236:9000/index.html`
2. 通过以下方式发送到手机：
   - 微信/QQ 发给自己
   - 支付宝"朋友"聊天
   - 邮件发送到手机
   - 短信发送
3. 在手机支付宝中打开链接

### 方法 3: 支付宝内搜索

如果已部署到有域名的服务器：
1. 在支付宝搜索你的小程序/生活号
2. 从菜单进入打卡页面

## 🔐 用户识别原理

```
用户打开链接
    ↓
检测是否在支付宝环境
    ↓
是 → 调用支付宝 JSAPI 获取 AuthCode
        ↓
    生成用户 ID: alipay_{authCode}
    ↓
否 → 生成游客 ID: guest_xxx
    ↓
查询/创建 Supabase 用户记录
    ↓
加载用户打卡数据
    ↓
显示个性化页面
```

## 📊 数据结构

### 用户表（users）
| 字段 | 说明 |
|------|------|
| alipay_user_id | 用户唯一标识（如：alipay_abc123） |
| points | 当前积分 |
| created_at | 创建时间 |

### 打卡记录表（checkins）
| 字段 | 说明 |
|------|------|
| alipay_user_id | 用户 ID |
| checkpoint_id | 打卡点 ID（1-5） |
| points | 获得积分 |
| checked_at | 打卡时间 |

## 🧪 调试技巧

### 1. 查看用户 ID

在支付宝打开后，摇一摇手机（或连接电脑调试），查看控制台：

```
🔐 支付宝环境：✅ 是
👤 用户 ID: alipay_AUTHCODE123
✅ Supabase 用户管理初始化完成
```

### 2. 查看数据库

在 Supabase 后台运行：
```sql
-- 查看所有用户
SELECT * FROM users ORDER BY created_at DESC;

-- 查看某用户的打卡记录
SELECT * FROM checkins 
WHERE alipay_user_id = 'alipay_xxx';

-- 查看排行榜
SELECT alipay_user_id, points 
FROM users 
ORDER BY points DESC;
```

### 3. 清除用户数据（测试用）

在浏览器控制台运行：
```javascript
// 清除当前用户的本地缓存
SupabaseManager.clearUserData();
location.reload();
```

## ⚠️ 注意事项

### 1. 支付宝权限

首次打开时，支付宝会提示"获取用户信息"，需要用户同意：
- ✅ 同意 → 获取真实用户 ID
- ❌ 拒绝 → 使用匿名 ID（alipay_anonymous_xxx）

### 2. 网络要求

- 手机和电脑必须在同一 WiFi 网络
- 如果使用局域网 IP（30.230.99.236）
- 生产环境需部署到公网可访问的服务器

### 3. 数据持久化

- ✅ 云端存储（Supabase）
- ✅ 本地缓存（localStorage）
- 即使清除浏览器缓存，云端数据不会丢失

### 4. 测试环境 vs 生产环境

**测试环境**：
- 使用局域网 IP
- 需要手机电脑同 WiFi
- 适合开发测试

**生产环境**：
- 部署到公网域名
- 使用 HTTPS
- 配置支付宝小程序/生活号

## 📱 常见问题

### Q: 为什么我的用户 ID 是 guest_xxx？
**A**: 说明不在支付宝环境打开。在支付宝中打开链接即可。

### Q: 多个用户数据串了？
**A**: 检查 Supabase 中是否使用 `alipay_user_id` 字段查询，确保没有使用固定的 userId。

### Q: 打卡后数据没有保存？
**A**: 
1. 查看浏览器控制台是否有错误
2. 检查 Supabase 连接是否正常
3. 确认 users 表中有用户记录

### Q: 如何知道当前是哪个用户？
**A**: 打开浏览器控制台，查看日志输出：
```
👤 用户 ID: alipay_xxx
```
